# 404agi-agent

Autonomous character engine for **404AGI** ($404) — the failed AGI meme coin.

This service is the *voice* of the project. It generates tweets in character,
answers chat / mentions in character, and reacts to wallet activity in
character — all gated by safety rails and a kill switch.

It does **not** post to X directly. Generated tweets are written to a local
JSONL log; the publishing layer (X API client or manual operator) is a
separate concern.

---

## Quick start

```bash
cd agent
cp .env.example .env
# fill in GEMINI_API_KEY (https://aistudio.google.com/app/apikey)

npm install
npm run typecheck      # sanity check
npm run test:prompts   # generate sample outputs (mock if no API key)
npm run dev            # start HTTP server on :8787
```

### Try it

```bash
# generate one tweet (random category)
npm run tweet:gen

# force a category, force a language
npm run tweet:gen -- --category=lore --lang=tr

# wallet event preview
npm run wallet:tweet -- --kind=self_buy --sol=0.05

# hit the response endpoint
curl -X POST http://localhost:8787/respond \
  -H 'content-type: application/json' \
  -d '{"channel":"web","message":"wen moon ser?"}'
```

## HTTP API

The web app and Telegram bot consume this service over HTTP.

### `POST /respond`

```json
{
  "channel": "web",          // "x" | "tg" | "web" | "internal"
  "message": "wen moon ser?",
  "user": "optional-id",
  "lang": "en",              // optional override; otherwise auto-detected
  "ephemeral": false         // if true, skip memory write
}
```

Returns the generated reply text, language, sanitization notes, timestamp.
On rate limit → `429`. On generation error → `500`. On banned content the
service returns a safe in-character fallback (200 OK).

### `POST /tweet/preview`

```json
{
  "category": "meme",        // optional — picks weighted-random if omitted
  "lang": "en",
  "marketBlurb": "optional context for trade tweets",
  "communityBlurb": "optional context for holder tweets",
  "bypassRate": false
}
```

Returns the generated tweet object. Always log-only — never posts.

### `POST /wallet/event`

```json
{ "kind": "self_buy", "sol": 0.05, "tx": "optional-signature", "note": "optional" }
```

Other kinds: `tip_received`, `milestone`, `balance_low`. See
`src/hooks/wallet.ts`.

### `GET /health`

Returns `killSwitch`, current rate-limit buckets, model selection. Used by
the main brain for status checks.

## Auth

In production set `AGENT_SHARED_SECRET` to any random string. Callers
(web app, TG bot) must send it as `X-Agent-Token` header. In dev (no secret
set) the endpoints are open for fast iteration.

## Safety model

Defense in depth — three independent layers:

1. **System prompt (`prompts/system.ts`)** instructs the model to refuse
   politics, religion, identity attacks, financial promises, self-harm, and
   to deflect *in character* rather than break voice.
2. **Output safeguards (`lib/safeguards.ts`)** are a regex-based deny list
   that runs on every generation. If the model slips through, the tweet is
   rejected; the response gets a fallback string in character. They also
   strip hashtags, excess emoji, markdown, and over-length tweets.
3. **Operational** — kill switch (`AGENT_KILL_SWITCH=1`) halts all
   generation immediately. Rate limits cap output at 4 tweets / 10 responses
   per hour by default (configurable). The main brain can flip the switch
   without touching code.

## Models

- `gemini-2.0-flash` — tweet generation (cheaper, fine for short outputs).
- `gemini-2.5-flash` — chat responses (better at conversational nuance).

Both selectable via env (`GEMINI_MODEL_TWEET`, `GEMINI_MODEL_CHAT`).

## Calibration

The hand-authored canonical samples live at
`samples/test_outputs.md`. They are the calibration target. Re-run
`npm run test:prompts` (with a Gemini key) to compare live outputs to the
canonical voice. The live-run file is written to `samples/live_run.md` and
gitignored.

If live outputs drift from canonical, the system prompt is what needs
fixing — not the model.

## Deployment

Designed to run on **Vercel** (serverless functions) or any Node 20+ host.

- `npm run build` → emits `dist/`
- `npm start` → runs `dist/server.js`
- For Vercel cron-driven tweet generation, wrap `generateTweet()` in a route
  handler. The publishing layer (X API or manual queue) is intentionally
  outside this service.

Memory and tweet log persistence currently use the local filesystem. For
multi-instance / serverless deploys, replace `lib/memory.ts` and the JSONL
appenders with Vercel KV or Redis — the function signatures are
deliberately minimal so a swap is a one-file change.
